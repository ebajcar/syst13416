Assignment 3 SYST13416 Introduction to Linux Operating System

Write a bash script called rmv (no extension) that will provide users with a safe way to delete files and directories by mimicking the trashcan in Windows. The user can run the utility with the following options:

rmv -d[elete] -e[mpty] -h[elp] -l[ist] file1 [file2 [file3 [file4]]]

In other words, the user can delete up to four (4) objects, empty the trashcan, get help, and display the listing of trashcan. Examples of how the user would run the command:

rmv –d  some/path/file1  file2  ../another/path/file3
rmv –e 
rmv –h
rmv -l

 

Your utility should create a hidden directory called trash, if it does not exist, in the user’s home directory.

You may wish to use the bash shell getopts comments to look at the command line for any options. If a matching letter is found by a case statement, the script commands up to the two semi-colons are executed.

Option d[elete]: When the user selects to delete up to four files, your utility will move those files from their current locations to the trashcan. Allow the user to provide pathnames to each file (in other words, the files do not need to be from the same directory). If your utility allows the user to specify regular files only, the maximum grade for the assignment can be 80%. If your utility handles the removal of directories in addition to regular files, the maximum grade can be 100% or higher.

Option e[mpty]: when the user selects to empty the trashcan, prompt the user with three (3) options:

Do you wish to empty the trashcan?
f) empty the trash
y) empty the trash inter-actively
n) abort emptying trash

Then your utility will empty the entire content of the trashcan, when user selects the option f; when the user selects option y, your utility will prompt the user for each file (and directory) and delete only those the user selects to delete; and abort the removal process when the user selects the option n.

Option l[ist]: When the user selects to list the content of the trashcan, provide the long listing of the trashcan. (If your utility handles directories, then your listing should include content of subdirectories). BONUS (5%): have your utility determine the length of the listing, it is longer than what fits on one screen, allow the user to page through the listing (back and forth).

Option h[elp]: When the user selects help option, or the user does not provide valid option or incorrect number of arguments, display message that contains information about the program, including name, author, version, short description, and instructions how to use the utility. Make it user friendly.

Consider when it is appropriate to clear the screen in your utility. This is part of making the utility easy to use.

Consider carefully what names you choose based on the Linux namespace and naming conventions.

In addition to the help documentation, make sure you document your code well, don’t forget to include prolog (header) and inline comments explaining how your code works.

Use proper indentation to make your code easy to read.
