Exercise 2: View The Calendar
  1. The cal command without any option or argument will output the current month's calendar. Try:
    ~$  cal
  2. You can specify the year as an argument. Try:
    ~$  cal 1897
  3. You can specify the month and year as arguments. Try:
    ~$  cal 2 1991
  4. To get information about other options and arguments for the command cal, examine its manual page:
    ~$  man cal
  5. How do you display the calendar for the current month?
  6. How do you display the calendar for the same month last year?
  7. How do you display the calendar for the year 1980?

Exercise 8: Communicating with Others
  1. The mesg command determines whether other users can send messages to you--and the write command can send a message from you to another user. Read the manual pages for mesg and write. Note the syntax of the command. Find a partner and send each other a message using write. What options and arguments did you use? The syntax of a write command line is
  2. ~$ write userid [terminal]
    write your message here
    more here...
  3. The userid is the user name and the terminal is optional.
  4. The terminal is optional, but useful if the person is logged in several times and you want to target a specific terminal.
  5. Each user has the option to deny or accept messages. To accept messages, run the command:
  6. ~$ mesg y
  7. To stop accepting messages, run the command:
  8. ~$ mesg n
  9. To establish two-way communication with another user,
    • you and the other user must each execute write, specifying the other's userid. The write utility then copies text, line by line, from one keyboard-display to the other.
    • Sometimes it helps to establish a convention, such as typing o for "over" when you are ready for the other person to type and typing oo for "over and out" when you are ready to end the conversation.
    • When you want to stop communicating with the other user, press  Control+D  at the beginning of a line, which tells the write to quit, displaying EOF on the other user's terminal, and returns you to the shell. The other user must do the same.
  10. The wall command allows you to send (broadcast) a message to everyone who is logged in on the system.
    DO NOT USE the wall command on atlas (or any other production machine) - you may be disrupting or distracting users that are trying to do their work!! Reserve the use of the wall command for administrative tasks, such as informing users of upcoming maintenance, etc.
  11. The mail command allows you to send email. Test it, by sending yourself an email to your Sheridan mail. Check your email to verify that it worked.
  12. ~$ mail userid@sheridanc.on.ca
    Subject: type your subject here Enter
    Type your message here Control+D
