    • Exercise 1: Read and Reflect

    To help you develop your critical thinking and system thinking, consider the following questions after reading the material for Module 2:

    1. What is a shell? What features does it provide?
    2. List the components of the bash programming language. Give example of each. How does it compare to Java and Javascript?
    3. What is redirection? pipelines? Explain.
    Exercise 2: Navigate FS with relative and absolute pathnames
    1. The directory in which you are working at the current time is called your "working directory" The command to display your working directory is pwd, which stands for "print working directory". Give the command to find out the directory path to your current location: 
      
        ~$ pwd
    2. Display the names of the objects in your current directory: 
      
        ~$ ls
    3. Display long listing of the current directory ("el es dash el", note: not ones) 
      
        ~$ ls -l
    4. List the contents of the current directory. (Note the ownership and group membership.)
    5. Change to the root directory of the Linux file system. (Note you start in your home directory.) 
      
        ~$ cd / 
        /$
    6. Using an absolute pathname, change to the mnt directory. (Note you are in / directory.)
    7. ~$ cd /mnt
      /mnt
    8. Change to the parent directory. (note you are back to the root directory)
    9. /mnt$ cd ..
      /$
    10. Using an relative pathname, change to the mnt directory.
    11. /$ cd mnt
      /mnt
    12. Using a relative path, change to the etc directory. (Note you are in /mnt directory, which is a sibling directory to /mnt.)
    13. ~$ cd ../etc
      /etc
    14. Give the following command and note where you end up.
    15. /etc$ cd
      ~$
    16. Change to /bin directory. From here, the /bin directory, continue to the next exercise.
    17. ~$ cd /bin
    Exercise 3: Navigate FS with shortcuts
    1. Change to your home directory using an absolute path
    2. Change to /etc directory
    3. Change to your home directory using a relative path
    4. Use the cd command to go to the /usr/bin directory.
    5. Use the cd command to return to your user's home directory.
    6. Again, use the cd command, but this time go to the system's root directory.
    7. Type the following command (symbol tilde) and note where you are.
    8. /$ cd ~
    9. Next, type the following command (symbol dash) and note where you are
    10. /$ cd -
    11. Change to your instructor's home directory
    12. ~$ cd ~bajcar
    13. Change to your instructor's public sub-directory
    14. ~bajcar$ cd public
    15. In public, change to datafiles sub-directory.
    16. ~bajcar/public$ cd datafiles
    17. Now, use the cd command to take you to your home directory
    18. ~bajcar/public/datafiles$ cd
    19. Next, type the following command (symbol dash) and note where you are.
    20. ~$ cd -
    21. What does the dash do? Change to a variety of different directories (explore!) Repeat this several time from different subdirectories until you find the pattern.
    22. Change to your home directory using the least number of keystrokes.

    Exercise 6: Create a Directory Sub-tree
    1. Make sure you are in the bash shell.
    2. Navigate to the temp directory created in the previous exercise.
    3. Display the names of the objects in your current directory.
    4. Create a subdirectory called usr
    5. ~$ mkdir usr
    6. 5. Change to the new subdirectory usr (Is this a relative or absolute pathname?)
    7. ~$ cd usr
      ~/usr$
    8. Create a subdirectory called staff
    9. ~/usr$ mkdir staff
    10. Create the following subdirectories. Note that we are remaining in the usr directory. (this is a single-line command, the text is wrapping!)
    11. ~/usr$ mkdir staff/adams staff/joan staff/brian staff/zeke staff/bill reports meetings
    12. Create the following directories. Note there are no spaces in argument one and no spaces in argument two. Check what was actually created. What do the braces do? (this is a single-line command, the text is wrapping!)
    13. ~/usr$ mkdir reports/rpt{A,AA,AB,C,3,75} notes{1,2,3,4,5}
    14. Using cd, ls, and pwd, navigate your newly created subdirectory, starting with usr as the top of your tree, and draw (on paper) the structure of the subtree.
    15. Once you have the drawing completed, explore the -R option of the ls command. Compare the result to your drawing.
